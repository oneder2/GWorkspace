# 更新日志

本文档记录项目的所有重要更新和变更。

## v2.7.0 (博客体验革命与社交传播增强 - 2026-02-26)

### 极致阅读交互
- **即时图片缩放**：集成 `medium-zoom` 库。现在点击文章图片可实现如 Medium 般丝滑的无缝放大体验，并辅以 12px 强度的毛玻璃遮罩背景。
- **Mac 风格代码块**：重构了代码渲染引擎。所有代码块现在拥有精美的深/浅色外壳、语言标识标签以及带反馈动效的一键复制功能。
- **智能目录导航 (TOC)**：实现了全自动的侧边栏目录生成。支持随动滚动高亮（精确到像素级判断）和平滑锚点跳转，大幅提升长文阅读效率。
- **微交互增强**：为博客列表引入了 **50ms 交错淡入动画** 和全新的**流光骨架屏 (Shimmer Skeleton)**，消除了内容加载瞬间的视觉跳变。

### 社交传播与分享
- **后端绘图引擎**：在后端引入 `node-canvas`，根据文章标题、分类、日期动态生成高清 OG 预览图（1200x630）。
- **一站式分享中心**：重构了分享逻辑。点击分享按钮将唤起集成了“动态海报预览”和“文章链接复制”的统一中心，支持带二维码的社交卡片生成与下载。
- **动态海报缓存**：实现了服务端图片缓存机制，首位访客触发生成后后续访问可瞬间响应，平衡了视觉美感与服务器性能。

### SEO 与数据闭环
- **Sitemap 实时化**：废弃了静态编译生成 Sitemap 的旧策略。现在由后端 API 根据数据库状态实时流式输出 `sitemap.xml`，确保搜索引擎索引永远与文章发布同步。
- **结构化数据增强**：完善了 JSON-LD 注入逻辑，确保每篇文章都包含精确的作者、发布/修改时间及目录指纹，提升搜索引擎摘要展示质量。

### 时间管理与 Bug 修复
- **时间精度重构**：全面修复了导致发布日期显示比设置日期“少一天”的时区偏差 Bug。现在全站时间支持精确到分钟的本地化显示。
- **数据一致性加固**：重写了后端 Blog 模型，确保在编辑文章时能正确保护原始发布日期（Published At），仅自动更新修改时间（Updated At）。
- **侧边栏功能整合**：将原本散落在底部的点赞、阅读量统计、预计阅读时间（基于字数计算）及标签云统一整合至右侧 Sticky 信息栏，实现信息密度最优布局。

## v2.6.0 (性能巅峰与生产级稳定性 - 2026-02-26)

### 图片上传与存储系统
- **全字符集支持**：修复了中文文件名导致的上传崩溃问题，现在完美支持 UTF-8 编码的文件名存储与预览。
- **上传限制提升**：将后端及 Nginx 转发层的文件大小限制从 5MB 提升至 **20MB**，满足高分截图上传需求。
- **稳定性修复**：解决了 R2 存储桶 SSL 握手失败、签名不匹配（SignatureDoesNotMatch）及 Content-Disposition 字符非法等一系列生产环境核心故障。

### 极致性能优化
- **PWA 本地秒开**：为云端图片引入了 **Cache-First** 运行时缓存策略。图片一旦加载过一次，后续预览将直接从本地硬盘读取，实现零延迟、零流量消耗。
- **上传乐观更新**：编辑器上传图片实现“秒开”预览。利用本地 Blob URL 机制，图片在上传瞬间即显示，不再等待云端往返。
- **防抖渲染引擎**：为 Markdown 预览引入 **300ms 防抖机制**，彻底消除了连续输入时的图片闪烁现象，显著降低了 CPU 解析负载。

### 管理后台 UI/UX 革命
- **架构层级提升**：重构了管理后台布局，使侧边栏在 `App.vue` 层级运行，解决了高度塌陷和滚动不一致的问题。
- **视觉完全对齐**：管理后台侧边栏现在 100% 套用主页面设计系统，废弃了低对比度的文字色，确保在各种主题下均清晰可见。
- **状态逻辑修复**：通过引入响应式状态管理，修复了侧边栏按钮点击后残留 hover 样式的顽固 Bug。

### 基础设施与自动化
- **CI/CD 自动化注入**：升级了 GitHub Actions 部署工作流，现在能自动将 GitHub Secrets 中的 R2 配置注入到自建服务器的 `.env` 文件中，实现了真正的自动化运维。
- **健壮性增强**：为天气接口添加了超时控制与异常兜底，防止外部 API 故障影响主站运行。

## v2.5.0 (主题系统重构与 PWA 增强 - 2026-02-24)

### 样式系统重构
- **语义化变量体系**：引入了全新的 CSS 变量架构（如 `--text-main`, `--bg-card`, `--border-base`），废弃了全站硬编码的 `slate` 颜色类，实现了真正的 UI 逻辑与视觉表现分离。
- **全站清理与适配**：完成了对首页、博客、工具箱、管理后台等所有核心页面的语义化重构，确保全站组件能完美适配任何自定义主题。
- **悬停可见性优化**：重新设计了全局交互的 `hover` 逻辑。悬停时不再降低背景不透明度，而是通过**同步提升背景浓度与毛玻璃模糊度**来增强视觉焦点，彻底解决了复杂背景下悬停文字难以辨识的问题。

### 主题自定义增强
- **智能色阶推导**：重写了 `useCustomTheme` 逻辑，引入颜色推导算法。用户只需选择一个主色，系统即可自动生成配套的深/浅色阶及最佳对比色。
- **交互调节能力**：在主题设置中新增了**背景透明度**和**毛玻璃强度**调节滑块，允许用户根据个人喜好微调 UI 的磨砂质感。
- **预设主题扩展**：新增了极光绿、深海蓝、夕阳橙、樱花粉等多款预设主题。

### PWA 应用化
- **支持浏览器创建应用**：重新引入并优化了 PWA 系统。通过配置 `VitePWA` 和注册 Service Worker，现在用户可以在支持的浏览器中将网页作为独立应用安装。
- **离线资源预缓存**：优化了 manifest 配置，确保核心图标、背景图和字体资源能够被有效缓存，提升二次加载速度。

### 功能修复与体验
- **滚动交互修复**：修复了博客页面、详情页及编辑器在滚动到底部时内容被遮挡或难以查看的问题，统一增加了底部缓冲内边距。
- **管理功能增强**：在博客编辑器中直接集成了“删除文章”功能，简化管理流程。
- **评论系统优化**：为评论列表添加了删除权限校验及多语言支持。

## v2.4.0 (性能优化与体验增强 - 2026-02-23)

### 性能优化
- **后端性能优化**：在敏感词过滤系统引入正则表达式缓存机制，极大降低高频请求下的 CPU 开销。
- **无需重启加载**：实现敏感词库热重载功能，支持通过管理 API 动态更新词库。

### 交互体验增强
- **复制反馈优化**：为编码工具 (Encoder) 和 Markdown 编辑器添加了 "Copied!" 视觉反馈和状态切换动画。
- **登录窗口层级修复**：使用 `Teleport` 技术将认证弹窗挂载至 `body` 顶层，彻底解决在复杂页面结构下的层级覆盖和遮罩显示异常。
- **天气定位修复**：修正了天气组件未使用经纬度参数的逻辑错误，确保天气数据与 IP 定位位置精准对齐。

### 架构与布局调整
- **移除 PWA 系统**：根据用户反馈移除了 Service Worker 相关的 PWA 自动更新和离线模式提示，简化项目架构并消除因提示框挤压导致的内容排版错乱。
- **首页布局重构**：恢复首页经典的居中对称布局，并在功能区下方新增了采用磨砂玻璃风格的"更新日志"区域。
- **开发环境配置**：创建 `.env` 规范，推荐使用 Vite 代理模式 (`/api`) 以解决本地开发跨域问题。

### 国际化支持
- **双语同步**：为首页更新日志区域同步添加了中英双语翻译。

## v2.2.0 (UI/UX优化版本 - 2026-01-XX)

### UI/UX 全面优化

- **背景蒙板优化**：
  - 优化了全局背景图片的遮罩层设计
  - 亮色模式使用浅色渐变（from-white/45 via-white/35 to-white/55），提高文本对比度
  - 暗色模式使用深色渐变（from-slate-900/75 via-slate-800/80 to-slate-900/85），确保浅色文字清晰可读
  - 改进了遮罩层的过渡动画，提供更流畅的视觉体验

- **文本颜色优化**：
  - 修复了HomePage主标题在背景图片上对比度不足的问题（从text-slate-200改为text-slate-800 dark:text-slate-100）
  - 优化了Header标题颜色，确保在所有背景下都有足够的对比度
  - 改进了PortfolioPage和BlogDetailPage中的标题颜色
  - 所有标题现在都符合WCAG对比度标准，提供更好的可读性

- **玻璃效果透明度优化**：
  - 优化了glass-main的背景透明度（从rgba(200, 200, 200, 0.35)改为rgba(255, 255, 255, 0.55)），提高文本可读性

  - 优化了glass-sidebar的透明度，确保导航文本清晰可读
  - 统一了各个玻璃效果类的透明度设计，提供更一致的视觉体验

- **视觉效果改进**：
  - 为主标题添加了drop-shadow效果，增强文本在背景上的可读性
  - 优化了悬停状态的透明度变化，提供更好的交互反馈
  - 改进了暗色模式下的背景遮罩，使用更深的渐变确保对比度

这些改进显著提升了项目的UI/UX质量，确保文本在所有背景下都有足够的对比度和可读性。
